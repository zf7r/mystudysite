01 Github Pages でサイト公開
  1. GitHub.comでリポジトリを作成
    1. github.com/new にアクセス 又は Home → Top Repositories の New
    2. リポジトリ名入力
    3. 他の項目は、デフォルトで可
    4. 左上にリポジトリが出ればOK

  2. リポジトリにアップロード
    1. Quick setup の 3行目の uploading an existing file をクリック
    2. ファイルをドラッグ＆ドロップ 又は choose your files でファイルを選択
      フォルダごと選択しないこと
    3. Drag additional ... の下に ファイルが表示されればOK
    4. Commit changes のボタンをクリック

  3. サイトの公開設定
    1. リポジトリの Settings タブをクリック
    2. サイドバーの Pages をクリック
    3. Branch で main /(root) として Save
    4. 進捗確認は、上にある Actions タブ（サイドバーのActionsではない）→
      pages build and deployment をクリック→
      Status Success になれば OK→
      Settings タブ → Pages クリックで戻る
    5. Visit site をクリックでサイト確認できる

  4. 編集(=再アップロード)の手順
    1. リポジトリの Code タブをクリック
      (緑のCodeボタンではない)
    2. +ボタンをクリック →
      Upload files をクリック
      (Upload files via upload ではない)
    3. 以下は、2のアップロードの(2)からと同じ

  5. 公開の一時停止
    1. リポジトリ→Settings タブ→Pages→
      三点メニューから Unpublish site をクリック
    2. サイトを再読み込みすると404エラーになる

  6. 一時停止からの再度公開
    1. リポジトリ→Actionタブ→
      一番上のpages build and deployment をクリック
    2. 右上に表示される Re-run all jobs をクリック
    3. 別ウィンドウが開くので、 Re-run jobs をクリック
    4. Status が Sucsess になるまで待つ。
    5. リロードするとサイトが復活している

  7. リポジトリ削除
    1. リポジトリ→Settings タブをクリック→
      一番下の Delete this repository をクリック
    2. 別ウィンドウで、I want to delete this repository をクリック
    3. 警告文が出て、To Confirm, type "ユーザー名/リポジトリ名"とあるので指示どおりタイプ
    4. Delete this repository をクリック
    5. リポジトリが削除される
